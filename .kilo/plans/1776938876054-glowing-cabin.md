## Step 4: Implementation

### 4.1 Request/Response Types

Create `src/domain/models/issuance.rs`:

```rust
// Request
#[derive(Debug, Deserialize)]
pub struct StartIssuanceRequest {
    pub offer: String,  // Raw offer (URI or JSON string)
}

// Response
#[derive(Debug, Serialize)]
pub struct StartIssuanceResponse {
    pub session_id: String,
    pub expires_at: String,  // ISO 8601
    pub issuer: IssuerSummary,
    pub credential_types: Vec<CredentialTypeDisplay>,
    pub flow: FlowType,
    pub tx_code_required: bool,
    pub tx_code: Option<TxCodeSpec>,
}

#[derive(Debug, Serialize)]
pub struct IssuerSummary {
    pub credential_issuer: String,
    pub display_name: Option<String>,
    pub logo_uri: Option<String>,
}

#[derive(Debug, Serialize)]
pub struct CredentialTypeDisplay {
    pub credential_configuration_id: String,
    pub format: String,
    pub display: CredentialDisplay,
}

#[derive(Debug, Serialize)]
pub struct CredentialDisplay {
    pub name: String,
    pub description: Option<String>,
    pub background_color: Option<String>,
    pub text_color: Option<String>,
    pub logo: Option<Logo>,
}

#[derive(Debug, Serialize)]
pub struct TxCodeSpec {
    pub input_mode: String,  // "numeric" or "text"
    pub length: Option<u32>,
    pub description: Option<String>,
}

// Error Response
#[derive(Debug, Serialize)]
pub struct IssuanceErrorResponse {
    pub error: &'static str,
    pub error_description: String,
}
```

### 4.2 Handler Implementation

Create `src/server/handlers/issuance.rs`:

```rust
pub async fn start_issuance(
    State(state): State<AppState>,
    Extension(tenant_id): Extension<Uuid>,  // From auth middleware
    Json(payload): Json<StartIssuanceRequest>,
) -> Result<impl IntoResponse, (StatusCode, Json<IssuanceErrorResponse>)> {
    // 1. Parse and validate the offer field
    //    - Try parsing as URI (openid-credential-offer://)
    //    - Try parsing as JSON string
    //    - Fetch credential_offer_uri if present
    
    // 2. Fetch issuer metadata from .well-known/openid-credential-issuer
    //    - Use Oid4vciClient from PR #159
    
    // 3. Fetch AS metadata if authorization_server is specified
    
    // 4. Determine flow from grants object:
    //    - If grants.authorization_code present → AuthorizationCode flow
    //    - If grants.pre-authorized_code present → PreAuthorizedCode flow
    //    - If no grants → default to AuthorizationCode (if AS supports)
    
    // 5. Extract tx_code from grants.pre-authorized_code.tx_code if present
    
    // 6. Generate session_id, create IssuanceSession with state AwaitingConsent
    
    // 7. Persist session using SessionStore (from PR #169)
    
    // 8. Build credential_types array by joining:
    //    - credential_configuration_ids from offer
    //    - credential_configurations_supported from issuer metadata
    //    - Use Accept-Language header for locale matching
    
    // 9. Return 201 with session info
}
```

### 4.5 Error Handling

Map errors to appropriate HTTP status codes:
- `400`: Invalid/malformed offer → `invalid_credential_offer`
- `502`: Cannot reach issuer metadata → `issuer_metadata_fetch_failed`
- `502`: Cannot reach AS metadata → `auth_server_metadata_fetch_failed`

## Acceptance Criteria Checklist

- [ ] Valid inline offer returns `201` with `session_id` and correct `flow`
- [ ] Valid `credential_offer_uri` offer returns `201` after fetching the URI
- [ ] Pre-authorized code offer with `tx_code` returns `tx_code_required: true` and populated `tx_code`
- [ ] Pre-authorized code offer without `tx_code` returns `tx_code_required: false` and `tx_code: null`
- [ ] Offer with no supported credential configuration IDs returns `400`
- [ ] Unreachable `credential_offer_uri` returns `502`
- [ ] Session is persisted and retrievable immediately after creation

## Design Decisions

1. **Oid4vciClient scope**: Singleton - create once at startup, share across all requests for better performance and connection pooling.
2. **Tenant validation**: Yes - validate tenant_id from JWT against the database before creating a session to prevent orphaned sessions.
3. **Branch name**: `feat/147-issuance-start-endpoint`
